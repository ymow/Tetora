Tetora の Discord 連携では、各スレッドを異なる agent にバインドすることで、複数の agent を**同時に**実行できます。各スレッドは独立したセッションを持ち、会話間で context が混入することはありません。

## 仕組み

メインの Discord チャンネルは単一の共有セッションを持ちます。異なる agent で並列タスクを実行するには、スレッドを作成して `/focus` で agent を割り当てます。

```
#general（メインチャンネル）              ← 共有セッション
  └─ Thread: "Refactor auth module"    ← /focus kokuyou → 独立セッション
  └─ Thread: "Write blog post"         ← /focus kohaku  → 独立セッション
  └─ Thread: "Competitor analysis"     ← /focus hisui   → 独立セッション
```

## 手順

**1. Discord スレッドを作成する** — メッセージを右クリック → スレッドを作成、または Discord のスレッドボタンを使用します。

**2. スレッド内で agent をバインドする：**

```
/focus kokuyou
```

バインドされると、そのスレッド内のすべてのメッセージが、独自の会話履歴を持つ指定 agent にルーティングされます。

**3. 他のタスクでも同様に繰り返す** — 必要な数だけスレッドを開き、それぞれに異なる（または同じ）agent を割り当てます。

**4. 完了したらバインドを解除する：**

```
/unfocus
```

## 設定

`config.json` でスレッドバインディングを有効にします：

```json
{
  "discord": {
    "threadBindings": {
      "enabled": true,
      "ttlHours": 24
    }
  }
}
```

## 注意事項

- **スレッドバインディングは期限切れになります** — デフォルトでは 24 時間後に失効します（`ttlHours` で変更可能）。期限切れ後、スレッドはメインチャンネルのルーティングに戻ります。
- **セッションは完全に隔離されています** — スレッドの context がメインチャンネルや他のスレッドに漏れることはありません。
- **同時実行の上限** — すべてのチャンネルとスレッドはグローバルの `maxConcurrent` 制限を共有します（デフォルトは 8）。上限を超えたメッセージはキューに入ります。
- `/focus` はスレッド内でのみ動作します。メインチャンネルは常に単一セッションを使用します——リセットするには `!new` を使ってください。
