Discord チャンネルのセッションは、時間の経過とともに context が蓄積されていきます。agent が混乱しているとき、または全く別のトピックに切り替えたいときは、`!new` でクリーンな状態から始めましょう。

## 使い方

Tetora が有効になっている任意のメイン Discord チャンネルで入力します：

```
!new
```

以上です。現在のセッションはアーカイブされ、次のメッセージから新しいセッションが始まります。

## リセットされるもの

- **会話の context** — このチャンネルでの過去のメッセージの記憶がない状態から agent が開始します。
- **セッション履歴** — 古いセッションは削除されずに履歴データベースにアーカイブされます。後から検索することも可能です。

## リセットされないもの

- **Agent の設定** — SOUL.md、config の設定、ロールの割り当ては変更されません。
- **メモリファイル** — agent の永続メモリは影響を受けません。
- **スレッドバインディング** — スレッド内の `/focus` バインディングは独立しており、影響を受けません。

## 使うべき場面

- agent が古い context を参照していたり、混乱しているように見えるとき
- 前の会話とは全く無関係の新しいタスクを始めるとき
- セッションが長くなりすぎて応答が遅くなっているとき
- 大きな変更（新しいデプロイ、config の更新）の後で、古い context が誤解を招く可能性があるとき

## !new と /focus の違い

| | `!new` | `/focus` |
|---|---|---|
| 使用場所 | メインチャンネル | スレッド内のみ |
| 効果 | セッションをアーカイブし、context をリセット | スレッドを特定の agent にバインド |
| スコープ | チャンネル全体に影響 | そのスレッドのみに影響 |

リセットではなく並列会話が必要な場合は、`/focus` を使ったスレッドを活用してください。
