隨著時間累積，Discord 頻道的工作階段會堆積大量 context。當 agent 出現混淆，或你正在切換至完全不同的主題時，`!new` 能給你一個乾淨的開始。

## 使用方式

在任何 Tetora 活動的主 Discord 頻道中輸入：

```
!new
```

就這樣。目前的工作階段會被封存，下一則訊息將開始全新的工作階段。

## 哪些內容會被重置

- **對話 context** — Agent 從零開始，不保留該頻道先前訊息的任何記憶。
- **工作階段歷史** — 舊的工作階段會封存至歷史資料庫，不會被刪除。你之後仍可搜尋它。

## 哪些內容不受影響

- **Agent 設定** — SOUL.md、config 設定與角色指派維持不變。
- **記憶檔案** — Agent 的持久記憶不受影響。
- **討論串繫結** — 討論串中的 `/focus` 繫結是獨立的，不受影響。

## 何時使用

- Agent 參照了過時的 context 或似乎感到困惑時
- 你要開始與先前對話完全無關的新任務時
- 工作階段已變得很長，回應速度變慢時
- 重大變更後（新部署、config 更新），舊 context 已不再適用時

## !new 與 /focus 的比較

| | `!new` | `/focus` |
|---|---|---|
| 使用位置 | 主頻道 | 僅限討論串 |
| 效果 | 封存工作階段、重置 context | 將討論串繫結至特定 agent |
| 範圍 | 影響整個頻道 | 僅影響該討論串 |

如果你需要的是並行對話而非重置，請改用討論串搭配 `/focus`。
